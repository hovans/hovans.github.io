tag:clash教程,clash使用教,clash-verge-rev使用教程

clash verge rev是一款使用tauri开发的clash GUI客户端，版本迭代很多次了，感觉已经很成熟了。优点：
1.开源，安全,使用tauri开发体积小，内存占用也小
2.支持windows、macOS、linux
3.自带支持中文
4.默认内核为clash meta，clash meta内核支持更多协议，比如vless/reality，也可以切换到clash-meta-alpha,可以支持hy2等
5.UI界面简洁，系统代理守卫，可定时重设系统代理。默认就是选择节点页面，默认主页可以更改。系统代理开关在设置页面上方。

适合使用习惯了clash for windows的用户，使用操作逻辑一样。

clash for windows删库停更新闻

clash-verge也不维护了。clash-verge-rev是新被接手维护的。

简单使用教程

1.下载安装

clash-verge-rev github下载地址 : 接手维护的。

官方TG群组：https://t.me/clash_verge_rev

m1,m2这些arm64的芯片的安装aarch64.dmg

原版本clash-verge github下载地址
clash备份下载地址

mac系统安装显示“文件已损坏” 原因是mac软件包没有经过开发者签名，请运到终端行命令授权：

更多问题官方解答：https://github.com/clash-verge-rev/clash-verge-rev/issues/125

2.切换为中文

settings–Verge Setting–Language选择为中文

3.导入订阅

一般机场都有一键导入订阅地址的。也可以手动复制订阅粘贴到clash verge中

以v2board面板机场为例子:点击【一键订阅】–【复制订阅地址】

点击【配置/订阅】–粘贴机场clash订阅链接到【配置文件链接】输入框中–然后点击【导入】–然后选中刚才下载的订阅文件

4.开启系统代理和开机自启

设置–系统设置–开启系统代理和开机自启

5.选择节点–开始使用

选择代理即可选择节点

一般情况下选择【规则】模式即可！

全局：无视所有规则，所有流量全部定向到全局分组中选择的节点，注意别全局又选择了直连节点
规则：默认走节点选择的节点，某些特定的节点会走特定规则组的节点，自动分流中国大陆和需要代理的网站（推荐使用该模式）
直连：无视所有节点，所有流量均不走代理

开启成功后【测试】页面，延迟应该都正常

如果超时 ，则说明没成功

到此，就可以翻墙了，大部分用户也只设置到这步即可。

高级设置

1.切换内核。可选操作

如果你想让你的clash 支持hy2。则需要切换内核，clash verge rev切换内核非常方便

设置–clash内核–点击clash-meta-alpha后选择授权，然后重启内核 即可

MacOS M芯片安装后请打开 终端 执行以下命令： sudo xattr -r -d com.apple.quarantine /Applications/Clash\ for\ Windows.app

2.Merge 合并处理(开启配置文件预处理)

可以让机场订阅更新时，不覆盖指定规则，比如机场的配置让aaa.com 走了代理，你需要不走代理。然后你更改了配置，但是更新订阅又变成走代理了，你希望更新订阅后你增加的规则不变

1)左边栏【订阅】–【新建】

2)类型选择[Merge]-名称随便写，描述可为空

3)右键merge那文件。选择编辑文件

4)编辑你需要增加的规则

比如下面的：
在规则最前面增加：
aaaaa.com 域名走直连
jichanggo.com走♻️ 手动切换，这里的”♻️ 手动切换” 需要看你的机场的是什么，有些默认是机场名，有些是”手动选择“，有些是”手动切换“

在规则最后面增加：bbbb.com域名不走代理

官方说明：
prepend-rules：类型和 clash rules 配置一致，内容合并到 rules 前
append-rules：类型和 clash rules 配置一致，内容合并到 rules 后
prepend-proxies：类型和 clash proxies 配置一致，内容合并到 proxies 前
append-proxies：类型和 clash proxies 配置一致，内容合并到 proxies 后
prepend-proxy-groups：类型和 clash proxy-groups 配置一致，内容合并到 proxy-groups 前
append-proxy-groups：类型和 clash proxy-groups 配置一致，内容合并到 proxy-groups 后

5)保存后右键启用Merge规则

6）给需要增加规则的机场，点击更新订阅–查看是否增加成功

一直往下翻，直到当前配置规则部分。成功过就有了。

如果是会写js脚本，还可以使用Script做加工。新建配置的时候选择scrip不是选择merge即可。其它操作一样。

比如：
删除 DOMAIN-SUFFIX,challenges.cloudflare.com,🧲 OpenAI 这条规则
在最后加上规则：DOMAIN,jichanggo.com,♻️ 手动切换

params是clash的object配置,params.rules就是rules部分、
config.proxies就是proxies部分
